# LOGIFLOW GPS 제거 및 출퇴근 시간 선택 설계

## 목표

출퇴근 등록에서 GPS와 시간 제한을 완전히 제거하고, 직원이 팝업에서 선택한 30분 단위 시간을 그대로 저장한다. 로그인, 중복 방지, 관리자 기능, 근태현황 및 근태 로그 구조는 유지한다.

## 범위

### GPS 제거

- 클라이언트 위치 권한 요청과 `navigator.geolocation` 호출을 제거한다.
- 직원 홈의 GPS 상태 UI와 관리자 운영 설정의 GPS UI를 제거한다.
- 서버 요청에서 위도, 경도, GPS 거리 및 인증값을 제거한다.
- 서버의 GPS 거리 계산과 반경 검증을 제거한다.
- `CONFIG`와 Script Properties 운영 설정에서 GPS 항목을 제거한다.
- 근태 로그 시트의 기존 컬럼 순서는 유지한다. GPS 거리와 인증 컬럼에는 빈 값을 기록해 기존 시트 및 대시보드 계약을 보호한다.
- 과거 로그 데이터는 수정하지 않는다.

### 시간 제한 제거

- 출근과 퇴근은 하루 24시간 등록 가능하다.
- 출근 가능 시간 정책 및 검증은 사용하지 않는다.
- 현재 활성 소스에는 `assertClockInRegistrationWindow()`와 `attendancePolicy`가 없으므로, 관련 코드가 다시 존재하지 않는지 정적 테스트로 검증한다.

### 시간 선택

- 출근 또는 퇴근 버튼을 누르면 기존 시간 선택 모달을 연다.
- 현재 시간을 다음 30분 기준으로 올린 값을 중심으로 이전 30분, 중심 시간, 다음 30분의 세 선택지를 제공한다.
- 예: `10:58`은 `10:30 / 11:00 / 11:30`, `18:12`는 `18:00 / 18:30 / 19:00`.
- 기본 선택은 가운데 시간이다.
- 서버는 선택된 시간을 다시 반올림하거나 버림하지 않고 그대로 근태현황에 저장한다.
- 선택한 시간의 날짜를 해당 근태 기록 날짜로 사용한다. 자정 전후 선택지 역시 선택된 날짜를 따른다.

## 데이터 흐름

1. 직원이 출근 또는 퇴근 버튼을 누른다.
2. 클라이언트가 세 개의 30분 단위 선택지를 표시한다.
3. 직원이 시간을 선택하고 등록을 누른다.
4. 클라이언트는 사번, 등록 유형, 선택 시간, 기기 정보만 서버로 전달한다.
5. 서버는 직원과 재직상태, 해당 날짜의 중복 등록 여부를 검증한다.
6. 서버는 선택 시간을 그대로 근태현황 시트에 저장한다.
7. 근태 로그에는 선택 시간과 기기 정보를 기록하고 GPS 두 컬럼은 빈 값으로 둔다.

## 유지되는 동작

- 사번 로그인과 자동 로그인
- 비밀번호 변경 및 초기화
- 관리자 권한과 직원 조회
- 출근 및 퇴근 중복 방지
- 근태현황 시트의 기존 컬럼 구조
- 근태 로그 시트의 기존 컬럼 구조
- 오늘 근무현황, 월별 근태 및 통계 조회
- 조출, 잔업, OT 계산 수식

## 오류 처리

- 시간을 선택하지 않은 경우 기존 Snackbar로 선택 안내를 표시한다.
- 존재하지 않는 사번, 퇴사자, 중복 등록, Sheets 오류는 기존 오류 흐름을 유지한다.
- GPS 관련 오류 문구와 오류 유형은 제거한다.

## 검증

- GPS 및 위치 권한 관련 실행 코드가 활성 파일에 남지 않았는지 정적 검사한다.
- 출퇴근 요청에 GPS 필드가 포함되지 않는지 검사한다.
- 선택한 시간이 서버에서 변환되지 않고 저장되는지 단위 테스트한다.
- `10:58`, `18:12`, 자정 전후 시간 선택지를 테스트한다.
- 출근 및 퇴근 중복 방지 계약을 검사한다.
- 기존 전체 정적 QA를 실행하고 GPS 관련 항목은 새 정책 기준으로 교체한다.
