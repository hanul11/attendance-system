# LOGIFLOW Apps Script 운영 구성

이 폴더는 LOGIFLOW 근태관리 웹앱의 Google Apps Script 운영 소스입니다.

## 연결 대상

- Spreadsheet ID: `1Zkm_mqrljBLFO_k2sAgCcgLWDKJhH01PqGK8MnGwOyE`
- 직원 기준 시트: `직원 사번 명단`
- 근태 운영 시트: `근태현황`
- 감사 로그 시트: `근태 로그`

시트 이름은 `Code.gs`의 `SHEET_NAMES`에서 관리하며, 모든 근태 읽기와 쓰기는
`CONFIG.attendanceSheetName`을 통해 `근태현황` 시트를 사용합니다.

## 운영 규칙

- 출근 및 퇴근 시간은 30분 단위 버림으로 저장합니다.
- 기존 `근태현황` 데이터 형식과 직원별 6개 컬럼 순서를 유지합니다.
- 출근, 퇴근, 중복 시도, GPS 실패 및 시스템 오류는 기존 `근태 로그`에 기록합니다.
- 직원 추가 시 `직원 사번 명단`을 기준으로 `근태현황` 직원 블록을 동기화합니다.

## 운영 점검

1. `SHEET_NAMES.attendance`가 `근태현황`인지 확인합니다.
2. `근태현황` 1행에 직원명, 2행에 `출근/퇴근/조출/잔업/OT/사용연차`가 있는지 확인합니다.
3. 출퇴근 등록 후 반환되는 `sheetName`이 `근태현황`인지 확인합니다.
4. 월별 조회, 통계 및 관리자 화면이 같은 운영 시트 데이터를 표시하는지 확인합니다.

실제 운영 기록 테스트는 지정된 테스트 사번과 일자를 사용해 진행하고, 인사팀 확인 없이
기존 직원의 운영 기록을 임의로 생성하거나 삭제하지 않습니다.
